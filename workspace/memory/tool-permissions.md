# Tool 权限分级体系

借鉴 Claude Code 的三级权限控制设计。

## 权限等级

| 等级 | 名称 | 权限范围 | 示例 |
|------|------|---------|------|
| **L1** | 只读 (Read) | 读取文件、查询数据、搜索 | `read`, `web_fetch`, `exec ls` |
| **L2** | 写入 (Write) | 创建/修改文件、安装依赖 | `write`, `edit`, `exec pip install` |
| **L3** | 执行 (Execute) | 运行命令、重启服务、发送消息 | `exec openclaw restart`, `message send` |

## 应用规则

### L1 - 只读（自动执行）
- 读取文件
- 搜索网络
- 查询数据
- 执行只读命令（ls, cat, grep）

### L2 - 写入（确认后执行）
- 创建/修改文件
- 安装 Python/Node 依赖
- 修改配置

### L3 - 执行（明确授权）
- 重启 Gateway
- 发送消息给用户
- 执行高风险命令

## 安全边界

### 禁止操作
- 删除系统文件
- 修改 /etc/ 下配置
- 安装来源不明的包
- 发送包含敏感信息的消息

### 敏感文件保护
- `openclaw.json` 中的 token 不可读取/展示
- `.env` 文件不可展示内容
- 用户密码/密钥不可记录到日志
