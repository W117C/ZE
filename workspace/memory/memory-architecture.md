# 三层记忆架构设计

借鉴 Claude Code 的三层记忆系统，结合 OpenClaw 特性设计。

## 架构

```
┌──────────────────────────────────────────────────────┐
│                  第一层：热记忆 (Hot)                   │
│  位置: memory/YYYY-MM-DD.md                          │
│  内容: 当日原始日志、操作记录、临时信息                  │
│  生命周期: 7 天后自动归档                              │
│  特点: 频繁读写，完整保留细节                            │
├──────────────────────────────────────────────────────┤
│                  第二层：温记忆 (Warm)                  │
│  位置: MEMORY.md                                     │
│  内容: 用户偏好、关键决策、经验教训、项目状态             │
│  生命周期: 30 天后评估是否保留                          │
│  特点: 结构化整理，AutoDream 定期更新                   │
├──────────────────────────────────────────────────────┤
│                  第三层：冷记忆 (Cold)                  │
│  位置: memory/archive/*.md                           │
│  内容: 历史事件归档、长期模式、已验证的知识               │
│  生命周期: 永久保留                                    │
│  特点: 只读参考，高度提炼                               │
└──────────────────────────────────────────────────────┘
```

## 数据流向

```
热记忆 (每日日志)
   │
   ▼ (AutoDream 整合，每 12h)
温记忆 (MEMORY.md)
   │
   ▼ (手动/自动归档，每月)
冷记忆 (archive/)
```

## 触发机制

### 写入触发
- **热记忆**: 每次对话/操作后自动追加
- **温记忆**: AutoDream 引擎自动整合
- **冷记忆**: 手动归档或月度自动整理

### 读取触发
- **热记忆**: 新 session 启动时读取今日 + 昨日
- **温记忆**: 新 session 启动时读取 MEMORY.md
- **冷记忆**: 按需搜索读取

## 容量控制

| 层级 | 大小限制 | 清理策略 |
|------|---------|---------|
| 热记忆 | 无限制 | 7 天后归档 |
| 温记忆 | 50KB | 30 天评估，删除过时内容 |
| 冷记忆 | 无限制 | 永久保留 |
